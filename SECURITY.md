# 安全说明

本文档说明z哥选股策略系统的安全特性、最佳实践和注意事项。

## 🔐 安全特性

### Token加密存储

#### 加密方式

- **算法**：SHA-256
- **存储格式**：哈希值
- **存储位置**：`config/token_config.json`

#### 安全机制

1. **哈希存储**：Token经过SHA-256哈希处理后存储，不可逆
2. **非明文保存**：配置文件中不包含原始Token
3. **验证机制**：验证时对输入Token进行哈希并与存储值比较

#### 代码示例

```python
from config import token_manager

# 保存Token（自动哈希）
token_manager.save_token("your_api_token_here", "描述")

# 验证Token
is_valid = token_manager.verify_token("your_api_token_here")

# 获取Token信息（不包含实际Token）
info = token_manager.get_token_info()
```

### Token验证

#### 格式验证

- 长度：16-64个字符
- 允许字符：字母、数字、- 和 _
- 正则表达式：`^[a-zA-Z0-9-_]{16,64}$`

#### 连接测试

- 尝试使用Token连接东财掘金API
- 验证Token权限和有效性
- 返回详细的错误信息

### 文件权限

#### 敏感文件

以下文件包含敏感信息，建议设置适当的文件权限：

- `config/token_config.json` - Token哈希存储
- `web/configs/weight_configs.json` - 权重配置
- `config/token_config.py` - 旧版Token配置（已弃用）

#### 推荐权限

```bash
# Windows
icacls config/token_config.json /inheritance:r
icacls config/token_config.json /grant:r "%USERNAME%:(F)"

# Linux/Mac（如果未来支持）
chmod 600 config/token_config.json
```

### 数据隔离

#### 用户数据

- 选股结果：存储在`reports/`目录
- 回测结果：存储在`backtest_output/`目录
- 缓存数据：存储在`cache/`目录

#### 缓存管理

- 自动清理过期缓存
- 支持手动清理
- 缓存文件格式：SQLite数据库

## ⚠️ 安全注意事项

### Token管理

#### ✅ 推荐做法

- 使用强Token（从东财掘金终端生成）
- 定期更换Token
- 不要分享Token给他人
- 使用描述性标签便于识别
- 及时删除不需要的Token

#### ❌ 禁止做法

- 不要在代码中硬编码Token
- 不要将Token提交到版本控制系统
- 不要通过不安全的渠道传输Token
- 不要使用过期或无效的Token
- 不要在公共网络下使用Token

### 网络安全

#### 连接要求

- 必须通过东财掘金终端连接
- 建议使用稳定的网络环境
- 避免使用公共Wi-Fi进行敏感操作

#### 数据传输

- 使用HTTPS协议（如果API支持）
- 验证SSL证书
- 注意网络钓鱼攻击

### 系统安全

#### 访问控制

- 限制对配置文件的访问权限
- 使用防火墙保护本地端口（8501、8502）
- 不要在共享计算机上运行

#### 日志记录

- 应用日志不包含敏感信息
- Token不会记录在日志文件中
- 定期检查日志文件

## 🚨 安全事件处理

### Token泄露

如果怀疑Token泄露：

1. **立即操作**：
   - 在东财掘金终端撤销Token
   - 生成新的Token
   - 在应用中更新Token

2. **后续措施**：
   - 检查使用日志
   - 确认没有未授权访问
   - 通知相关人员（如果适用）

3. **预防措施**：
   - 加强文件权限
   - 修改访问密码
   - 启用两步验证（如果可用）

### 系统入侵

如果发现系统被入侵：

1. **隔离系统**：
   - 立即断开网络连接
   - 停止所有服务
   - 不删除任何文件

2. **调查**：
   - 检查日志文件
   - 分析入侵路径
   - 确定影响范围

3. **恢复**：
   - 从备份恢复
   - 修改所有密码和Token
   - 更新系统到最新版本

4. **预防**：
   - 加强安全措施
   - 进行安全审计
   - 培训安全意识

## ❓ 常见问题

### Q1: Token加密后是否安全？

**A**：SHA-256是单向哈希算法，无法从哈希值反推原始Token。但如果攻击者获得哈希值，可以使用彩虹表暴力破解。因此建议：
- 使用强Token
- 定期更换Token
- 保护配置文件不被访问

### Q2: 为什么不使用更强的加密算法？

**A**：当前版本使用SHA-256哈希，主要考虑：
- 平衡安全性和性能
- 简化实现复杂度
- 避免管理密钥的复杂性

未来版本可能支持更高级的加密算法（如AES-256）。

### Q3: Token存储在本地是否安全？

**A**：本地存储的安全性取决于：
- 操作系统的安全性
- 文件权限设置
- 物理访问控制

建议：
- 使用安全的操作系统
- 设置强密码
- 限制他人访问您的计算机

### Q4: 是否支持多用户？

**A**：当前版本设计为单用户使用。多用户支持需要：
- 用户认证系统
- 权限管理
- 数据隔离

这是未来可能的增强功能。

### Q5: 日志文件是否包含敏感信息？

**A**：不会。应用日志中：
- 不记录Token（原始或哈希）
- 不记录敏感配置
- 仅记录操作状态和错误信息
