# QuantScout量化选股系统 - 项目解说

## 🌟 项目背景

QuantScout是一个基于东财掘金量化平台的智能选股与参数优化系统，旨在为量化初学者提供一个完整的学习实践案例。本项目展示了量化策略开发的完整流程，从策略逻辑实现到参数优化，再到回测验证。

**设计理念：**
- **模块化架构**：代码结构清晰，便于学习和扩展
- **策略-回测-优化闭环**：完整的量化开发流程
- **多算法支持**：暴力搜索、遗传算法、粒子群算法
- **可视化界面**：直观的操作体验

## 🎯 策略逻辑剖析

### 多维综合策略简介

多维综合策略是QuantScout的核心策略，融合了三大技术信号：

1. **KDJ指标** - 超买超卖信号
2. **知行趋势线** - 趋势跟踪信号
3. **深V信号** - 反转信号

### 指标详解

#### 1. KDJ指标

KDJ是经典的超买超卖指标，由K线、D线、J线组成：

```python
# KDJ计算逻辑
# J值 = 3*K - 2*D
# 当J值 < 20时，视为超卖信号
# 当J值 > 80时，视为超买信号
```

**选股条件**：J值 < 20（超卖区域）

#### 2. 知行趋势线

知行趋势线是一种趋势跟踪指标，通过白线和黄线的交叉判断趋势方向：

```python
# 趋势信号判断
# 白线斜率 > 0: 上升趋势
# 白线斜率 < 0: 下降趋势
# 白线上穿黄线: 金叉买入信号
```

**选股条件**：白线斜率 > 0（上升趋势）

#### 3. 深V信号

深V信号识别股票的反转形态：

```python
# 深V反转特征
# 开盘价 > 收盘价（阴线）
# 最低价接近前一根K线最低价
# 收盘价接近前一根K线开盘价
# 成交量放大
```

**选股条件**：出现深V反转信号

### 综合评分机制

系统采用权重评分机制，综合评估各指标：

| 指标 | 默认权重 | 说明 |
|------|---------|------|
| kdj_j | 25 | KDJ的J值 |
| trend | 25 | 知行趋势指标 |
| deepv | 10 | 深V反转信号 |
| volume | 8 | 成交量指标 |
| fundamental | 8 | 基本面指标 |
| position | 4 | 持仓比例 |
| risk_reward | 20 | 风险收益比 |

## 🔧 参数优化算法原理

### 支持的优化算法

#### 1. 暴力搜索

**原理**：遍历所有可能的参数组合

**优点**：保证找到全局最优解

**缺点**：计算量大，耗时较长

**适用场景**：参数空间较小的场景

#### 2. 遗传算法

**原理**：基于生物进化原理，通过选择、交叉、变异迭代优化

**算法流程**：
```
初始化种群
    ↓
评估适应度
    ↓
选择（保留优秀个体）
    ↓
交叉（基因重组）
    ↓
变异（随机变异）
    ↓
迭代直到收敛
```

**优点**：快速收敛，适合大规模参数空间

**缺点**：可能陷入局部最优

#### 3. 粒子群算法

**原理**：模拟鸟群觅食行为，通过个体和群体经验迭代优化

**算法流程**：
```
初始化粒子群
    ↓
更新粒子速度（基于个人最优和全局最优）
    ↓
更新粒子位置
    ↓
评估适应度
    ↓
更新个人最优和全局最优
    ↓
迭代直到收敛
```

**优点**：收敛速度快，参数调节简单

**缺点**：对初始参数敏感

## 📊 实际使用案例

### 案例1：快速选股

**场景**：投资者希望快速筛选出符合条件的股票

**操作流程**：
```
1. 启动策略控制器
2. 配置Token（首次使用）
3. 使用默认权重配置
4. 设置筛选参数（最大结果数、跳过ST股）
5. 点击「开始选股」
6. 查看选股结果
```

**预期结果**：系统返回综合评分最高的股票列表

### 案例2：参数优化

**场景**：投资者希望找到最优的策略参数组合

**操作流程**：
```
1. 启动参数优化器
2. 选择优化算法（推荐遗传算法）
3. 设置参数范围（止盈/止损比例、权重步长）
4. 生成参数组合蓝图
5. 点击「开始优化」
6. 查看优化结果
7. 发送最优参数到策略控制器
```

**预期结果**：系统返回收益率最高的参数组合

### 案例3：完整回测

**场景**：投资者希望验证策略的历史表现

**操作流程**：
```
1. 使用策略控制器选股
2. 对选股结果进行回测
3. 配置回测参数（初始资金、回测天数）
4. 执行回测
5. 查看回测指标和图表
6. 根据回测结果调整策略参数
```

**预期结果**：系统生成详细的回测报告
