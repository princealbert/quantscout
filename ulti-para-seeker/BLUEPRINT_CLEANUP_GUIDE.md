# 蓝图清理功能说明

## 概述

蓝图清理功能用于删除低质量的参数组合,保留最优组合,减小蓝图文件大小,提高系统性能。

## 功能位置

### 1. 蓝图管理页面 (手动清理)

在参数优化器的"蓝图管理"标签页,新增了"清理蓝图"功能:

**功能特性:**
- 选择要清理的蓝图文件
- 显示当前蓝图状态统计(总组合数、已完成、失败)
- 自定义清理参数:
  - 保留的最大总组合数 (默认1000)
  - 保留的最优组合数 (默认500)
  - 是否保留失败的组合
- 显示清理前后对比
- 自动归档被删除的组合

**使用步骤:**
1. 进入"蓝图管理"标签页
2. 找到"清理蓝图"部分
3. 选择要清理的蓝图文件
4. 查看当前状态统计
5. 调整清理参数(可选)
6. 点击"清理选中的蓝图"
7. 二次确认后开始清理
8. 查看清理结果

### 2. 运行优化页面 (自动清理)

在"运行优化"标签页,回测完成后会自动触发清理:

**触发条件:**
- 蓝图组合数超过1000个
- 回测全部完成后

**清理参数:**
- 保留的最大总组合数: 1000
- 保留的最优组合数: 500
- 不保留失败的组合

**显示信息:**
- 清理前组合数
- 清理后组合数
- 减少的组合数
- 归档信息(如有)

## 清理规则

### 保留策略

清理器按以下优先级保留组合:

1. **精英组合** (已完成 + 高质量)
   - 按收益率排序
   - 保留前N个(默认500个)
   - 收益率范围: -50% ~ 150%

2. **其他已完成组合**
   - 如果精英组合不足,从剩余已完成组合中补充
   - 直到达到总组合数上限(默认1000个)

3. **待处理组合** (pending)
   - 如果已完成组合不足,从pending中补充
   - 随机选择

4. **失败组合** (failed)
   - 默认不保留
   - 可通过参数选择保留

### 归档策略

被删除的组合会自动归档到:
- 文件名: `parameter_blueprint_archive_YYYYMMDD.json`
- 保存完整的组合和结果数据
- 可用于后续分析

## 使用建议

### 何时需要清理

- 蓝图文件超过1000个组合
- 蓝图文件大小超过50MB
- 系统性能明显下降
- 需要归档历史测试结果

### 清理参数建议

**保守策略** (保留更多组合):
- 最大总组合数: 2000
- 最优组合数: 1000
- 保留失败组合: 是

**标准策略** (默认):
- 最大总组合数: 1000
- 最优组合数: 500
- 保留失败组合: 否

**激进策略** (保留最少组合):
- 最大总组合数: 500
- 最优组合数: 250
- 保留失败组合: 否

### 注意事项

1. **不可逆操作** - 清理删除的组合无法直接恢复
   - 如需恢复,查看归档文件
   - 或重新生成参数组合

2. **保留精英组合** - 清理不会删除高质量的组合
   - 高质量定义: 收益率在-50%~150%之间
   - 按收益率排序后保留最优N个

3. **归档文件** - 被删除的组合会自动归档
   - 归档文件保存在同一目录
   - 包含完整的测试结果
   - 可手动导入或分析

## 技术细节

### 实现模块

清理功能使用 `ulti-para-seeker/utils/blueprint_cleaner.py` 实现:

**主要方法:**
- `clean_blueprint()` - 清理蓝图主方法
- `_filter_elite_combinations()` - 筛选精英组合
- `_remove_duplicates()` - 去重
- `_archive_removed_combinations()` - 归档

**参数说明:**
```python
BlueprintCleaner(
    max_total=1000,      # 保留的最大总组合数
    max_elite=500,        # 保留的最优组合数
    keep_failed=False      # 是否保留失败组合
)
```

### 调用方式

**自动清理** (app.py 第750-796行):
```python
cleaner = BlueprintCleaner(max_total=1000, max_elite=500, keep_failed=False)
cleaned_blueprint, archive_data = cleaner.clean_blueprint(
    blueprint_data,
    blueprint_file=blueprint_path,
    auto_archive=True
)
```

**手动清理** (app.py 第499-583行):
```python
cleaner = BlueprintCleaner(
    max_total=用户设置的值,
    max_elite=用户设置的值,
    keep_failed=用户选择
)
cleaned_blueprint, archive_data = cleaner.clean_blueprint(...)
```

## 故障排查

### 清理失败

**可能原因:**
1. 蓝图文件被占用
2. 磁盘空间不足
3. 归档文件已存在

**解决方案:**
1. 关闭所有正在运行的优化任务
2. 检查磁盘空间
3. 删除旧的归档文件
4. 在"蓝图管理"页面手动重试

### 归档失败

**可能原因:**
1. 目标目录无写权限
2. 磁盘空间不足
3. 文件名冲突

**解决方案:**
1. 检查目录权限
2. 手动删除旧归档文件
3. 使用不同的归档文件名
